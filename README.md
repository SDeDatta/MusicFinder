MUSIQ is a Java program used for music discovery to allow users to type nuanced natural language queries and find music that meets the criteria. Most music platfroms recommend music using listening history, but this project is designed to recommend music for users who like aspects of a song but want to adjust the vibe slightly. The program uses existing Kaggle datasets that have audio features attached to every song. Then, the program weights the different audio features using an API call to OpenAI's LLM to come up with a set of recommendations from the 90,000+ songs. All the songs are inputted into a graph, and connections between songs only arise if their audio features and genre are similar enough. Additionally, songs are only recommended if the user's query is considered reasonable. Meaning, a user will not use a reggae song as the seed and request for an EDM vibe. Additional query features include same-artist filtering, cross-language recommendations, and a language match toggle.
